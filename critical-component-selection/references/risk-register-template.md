# Risk Register Template

Every risk must have a mitigation action. A risk row without mitigation, owner, and review date is not complete.

Probability and impact values must be filled as `L`, `M`, or `H` before output is considered complete. Do not leave default-looking values in a final risk register.

Required risk categories:

- Supply / allocation / lead time
- Lifecycle / EOL / PCN
- Package / pinout / footprint
- SI / PI / thermal / mechanical
- Firmware / logic / toolchain
- Cost / MOQ / MPQ / commercial terms
- Substitute / second-source path
